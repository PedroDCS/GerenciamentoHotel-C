# GerenciamentoHotel-C
Trabalho feito utilizando a linguagem C, para a disciplina de Algoritmos II


Este relatório contém informações sobre o planejamento, execução e conclusão do trabalho realizado pelos estudantes Pedro Daniel Camargos Soares e Lucas Gabriel De Almeida, como requisito para obtenção de nota na disciplina de Algoritmos II.
O sistema tem o propósito de servir de gerenciamento de um hotel, desde cadastro de hospedes até sistema de reservas.
O mesmo ainda conta com um sistema de compras, vendas, checkin e checkout, caixa, dentre outros módulos.

O trabalho consistia em desenvolver um sistema de controle gerencial de um hotel, possuindo um total de cinco módulos, sendo eles:

2.1 Módulo de Cadastro e Gestão de dados 
Módulo de Cadastro e Gestão de dados, onde era necessário desenvolver um código em que o usuário poderia inserir, alterar e excluir dados de todas as categorias, sendo elas: Dados do hotel, dados dos hóspedes, dados das acomodações, dados dos produtos e dados dos fornecedores.

2.2 Módulo de Reservas
Módulo de Reservas, onde era necessário desenvolver um código em que o usuário poderia realizar ou cancelar uma reserva. Este módulo é sequencia direta do primeiro, pois é necessário pesquisar as acomodações registradas no sistema e registrar qual hóspede ocupará a mesma.
Este módulo foi difícil, pois uma parte importante da reserva era o desenvolvimento de um sistema de datas, além de ser o módulo que mais ocorreram bugs.
2.3 Módulo de Transações
Módulo de Transações, onde era necessário desenvolver um código em que o usuário poderia administrar o sistema, como controle do caixa, contas a pagar/receber e check-in/out.
Este módulo foi o que deu mais trabalho, pois foi necessário introduzir o sistema de check-in/out do hotel. A parte de realizar pagamentos com dinheiro ou cartão também foi difícil, pois foi necessário desenvolver vários arquivos para armazenar as compras realizadas e o histórico de compras, e por fim gerar uma nota fiscal. 

2.4 Módulo de Feedback
Módulo de Feedback, onde era necessário desenvolver um código em que o usuário pudesse gerar relatórios, sejam eles em tela ou em arquivo.
Este módulo foi fácil e difícil ao mesmo tempo, pois alguns relatórios como os de hóspedes e acomodações foram fáceis de fazer, já aqueles que envolviam dinheiro foram mais complicados. Determinados filtros, como os de código, foram fáceis de fazer, mas outros filtros como os de data foram mais complicados de fazer.
2.5 Módulo de importação/exportação de dados
Módulo de importação/exportação de dados, onde era necessário desenvolver um código em que o usuário pudesse importar/exportar dados para outras aplicações.
No fim esse módulo não foi tão difícil, a parte de exportar dados é bem fácil, já a parte de importar dados foi um pouco mais complicada por ter que ler um arquivo e selecionar o que seria importado.


3. 	CONSIDERAÇÕES FINAIS 
 
 
O trabalho realizado pode aprimorar as habilidades dos alunos no que diz respeito aos conhecimentos adquiridos durante o semestre, além de proporcionar a descoberta de coisas que não foram apresentadas em sala de aula.
O principal obstáculo encontrado foi a implementação de vários módulos diferentes que conversavam entre si, pois os mesmos costumavam apresentar bugs de compatibilidade no começo do projeto, mas com um pouco de pesquisas essas dificuldades foram deixadas para traz.
 No fim o trabalho se provou desafiador e mais difícil do que imaginado, porem foram aprendidos vários novos comandos em c, também a lidar melhor com strings e arquivos, além de ao lidar com os constantes bugs, resolvê-los e como evita-los.
Com o fim deste trabalho, foi ganha experiência em relação ao desenvolvimento de sistemas mais complexos, como para o mercado de trabalho, pois existe uma ideia de como funciona um trabalho deste nível.
